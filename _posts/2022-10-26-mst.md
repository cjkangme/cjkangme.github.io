# 최소 신장 트리란?

![MST](mst.webp)

## 연결 그래프

최소 신장 트리를 알기 위해서는 먼저 연결 그래프를 알아야한다.<br>
연결 그래프는 정점(vertax)과 간선(edge)이 있으며, 모든 정점을 간선으로 연결한 그래프를 말한다.

## 신장 트리(Spanning Tree)

신장 트리는 그래프의 모든 정점을 연결하면서 사이클이 없는 그래프를 의미한다.<br>
즉 간선의 수가 최소가 되는 그래프를 말하며, 이때 간선의 수는 항상 `정점의 수 - 1` 이다.<br>
하나의 그래프에서 신장 트리는 여러종류가 존재할 수 있다.

## 최소 신장 트리 (Minimum Spanning Tree)

마지막으로 최소 신장 트리는 각각의 간선에 가중치(비용)를 둔다.<br>
이 때 모든 정점을 가장 최소의 비용으로 연결한 그래프가 바로 최소 신장 트리가 된다.

최소 신장 트리를 구하는 대표적인 알고리즘을 몇 가지 배워보자.

# Kruskal MST 알고리즘

Kruskal MST 알고리즘은 Greedy method를 이용하는 방법이다.

Greedy method란 탐욕적으로 눈 앞의 이익만 좇는 방법을 말한다.<br>
전체 상황(문제)를 고려하지 않고 현재 주어진 상황만을 최적으로 해결한다면, 그것이 곧 전체에 대한 최적해가 될 것이라는 가정을 기반으로 한다.

## 과정

1. 그래프 간선의 가중치를 오름차순으로 정렬한다.
2. 가중치가 최소인 간선을 선택한다.
3. 해당 간선이 사이클을 이루지 않을 경우 MST 집합에 추가한다.

# 다익스트라(Djikstra) 알고리즘

다익스트라 알고리즘은 **음의 가중치가 없는 그래프**의 최단 경로 탐색 알고리즘이다.

최소 신장 트리를 구하는 것보다는 출발 정점(노드) x에서 도착 정점(노드) y까지의 최단 경로를 구하는 문제에서 활용할 수 있다.

## 과정

1. 출발 노드의 값을 0으로 설정하고, 이외의 노드는 모두 INF(무한대)로 설정한다.
2. 현재 노드로 출발 노드를 설정하고, 도착 노드를 설정한다.
3. 현재 노드의 값 + 간선 가중치를 이웃한 노드의 값과 비교한다.
4. 만약 현재 노드의 값 + 간선 가중치가 더 작다면 이웃한 노드의 값을 이 값으로 갱신한다.
5. 현재 노드의 모든 이웃 노드에 대해 3 ~ 4번을 반복한다.
6. 현재 노드의 상태를 Finished로 변경한다. (더이상 사용하지 않음)
7. 이웃한 노드 중 Finished 상태가 아니면서, 값이 제일 작은 노드(현재 최단거리인 노드)를 현재 노드로 설정한다.
8. 도착 노드가 Finished 상태가 될때까지 3 ~ 7번을 반복한다.
9. 도착 노드의 값이 바로 출발 노드부터 도착 노드까지의 최소 가중치가 된다.

# 벨만-포드 알고리즘

**음의 가중치가 있는 그래프**에서 사용할 수 있는 최단 경로 탐색 알고리즘이다.

## 다익스트라 알고리즘과의 차이점

- 다익스트라 알고리즘 : 현재 최단거리인 노드만을 선택하므로, 선택되지 않은 노드 중 음의 가중치를 갖는 간선을 무시할 수 있다. 즉, 음수 간선이 있을 때 최적의 해를 찾지 못할 수도 있다.

- 벨만-포드 알고리즘 : 매 단계마다 모든 노드를 선택해 최단 거리를 구한다. 즉 음의 가중치가 있더라도 최적의 해를 항상 포함하게 된다. 대신 그만큼 시간 복잡도가 증가한다.

## 과정

1. 출발 노드의 값을 0으로 설정하고, 이외의 노드는 모두 INF(무한대)로 설정한다.
2. 현재 노드로 출발 노드를 설정하고, 도착 노드를 설정한다.
3. 현재 노드의 값 + 간선 가중치를 이웃한 노드의 값과 비교한다.
4. 만약 현재 노드의 값 + 간선 가중치가 더 작다면 이웃한 노드의 값을 이 값으로 갱신한다.
5. 모든 노드에 대해 2 ~ 4번을 반복한다. (인접 노드로 이동하고, 인접 노드가 없을 경우 탐색을 종료)
6. 탐색 횟수가 `정점의 수 - 1`될 때까지 3 ~ 5번을 반복한다.
7. 6번을 1회 더 반복하고, 값이 갱신되는지 확인한다. (음의 순환 체크)
8. 만약 7번에서 값이 갱신되지 않는다면 도착 노드의 값이 최소 가중치이다.

## 음의 순환

음의 순환은 최단거리가 무한히 감소하는 순환을 말한다.<br>
사이클의 가중치 합이 음수일 때 발생한다.

비유하자면 A->B로의 가중치가 5, B->C의 가중치가 -3, C->A의 가중치가 -4라면, A->B->C를 무한히 한번 씩 순회할 때마다 가중치가 2씩 감소하게 된다.<br>
이럴 경우 알고리즘은 무한 루프에 빠지고 최저 가중치 역시 구할 수 없게 된다.<br>
때문에 한번 더 순회를 반복하여 음의 순환이 존재하는지 확인하는 것이다.
