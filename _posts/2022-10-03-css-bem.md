## 1. BEM이란

BEM은 보다 '읽기 쉬운' CSS를 만드는 방법론으로, 클래스의 작명 규칙을 정한 것이다.<br>
읽기 쉬운 CSS는 추후 쉽게 유지보수가 가능하고, 재사용이 용이하다는 장점을 갖는다.<br>

<br>

각각 여기서 BEM은 Block, Element, Modifier의 앞글자를 딴 말로, 이 3가지 항목으로 구성된 클래스명을 짓는다는 것을 뜻한다.

<br>

## 2. 기본 사항 / Block / Element / Modifier

### 기본 사항

BEM은 우선적으로 다음과 같은 원칙으로 작성된다.

- id명을 사용하지 않고, class명 만을 사용한다.
- 소문자와 숫자만을 이용하여 작명한다.
- 여러 단어의 조합은 단일 하이픈(`-`)으로 연결한다.

### Block

- block은 재사용이 가능한 독립적인 페이지 구성요소를 말한다.
- block의 이름은 해당 요소의 사용 목적을 나타낸다.
- block은 주변 환경에 영향을 받지 않아야 하므로, `margin`이나 `position`과 같은 속성을 설정하면 안된다.

> 예시 : menu, button, status-bar, login-form 등

### Element

- element는 block의 특정 기능을 담당하는 부분을 말한다.
- block에 의존하는 형태를 갖고 있어, 블럭 안에서 분리되면 사용할 수 없다.
- 더블언더바(`__`)를 이용하여 block-name_element-name과 같은 형태로 작성한다.
- element는 필수가 아니므로 element가 없는 block이 있을 수 있다.

> 예시 : menu\_\_title, button\_\_text, status-bar\_\_item, login-form\_\_btn 등

### Modifier

- modifier는 block이나 element의 형태, 상태 또는 동작을 말한다.
- 더블하이픈(`--`)을 이용하여 block이나 element 뒤에 사용한다.
- 불리언 타입, 키-밸류 타입 두가지 타입으로 구분할 수 있다.
  - 불리언 타입 : 수식어의 값을 true라고 가정한다.
    > 예시 : login-form_input&#45;&#45;disabled, button&#45;&#45;focused 등
  - 키-밸류 타입 : 성질이나 속성값을 나타낸다.
    > 예시 : menu&#45;&#45;theme-dark, button&#45;&#45;color-red

<br>

## 3. 혼합하기

하나의 요소에 여러가지 클래스명을 혼합하여 사용할 수 있다.<br>
예시

```html
<!-- `header` block -->
<div class="header">
  <!--
        The `search-form` block is mixed with the `search-form` element
        from the `header` block
    -->
  <div class="search-form header__search-form"></div>
</div>
```

이 경우 `search-form` block을 독립적으로 유지하고, `header&#95;&#95;search-form` element에 margin이나 position을 지정할 수 있다.<br>

<br>

## 4. 정리

이제 갓 배운 개념이고, 아직 본격적으로 활용하지 않아 얼마나 잘 사용할 수 있을지는 모르겠다.<br>
하지만 아주 간단해 보이는 사이트를 배끼는 과정에서도 클래스명, id명이 혼재되어 뒤죽박죽이 되는게 상당한 골치였다는 것을 생각하면, 이렇게 분류하는 방법이 말 그대로 '읽기 쉬운' CSS를 만드는 데 최적화 되어있는 것 같다.<br>
앞으로 HTML/CSS를 작성할 때 최대한 BEM 구조를 생각하며 작성해야겠다.
